# INDEX — Back-Of-Book Topical Index

Alphabetical "where is this covered?" index for the whole library. Each entry points a topic at the book(s) and section(s) that cover it, so an agent (or human) can jump from a term to the page without scanning. Keep it sorted; add compound topics as nested bullets under their head term.

> **Format.** One bullet per topic: `- Topic — Book NN §X`. Group alphabetically under single-letter `## A`, `## B`, … headers. For a topic with sub-aspects, nest the aspects as indented bullets. Cite at the most specific section you can. List multiple locations comma-separated when a topic genuinely spans books. Write "Book NN" as plain text — never as a link.

## Thematic clusters

A short second pass that groups related topics by theme for readers who think in areas rather than words. Optional but useful once the library is large.

<!-- EXAMPLE — replace -->
- **Getting started:** see [STARTER_KIT.md](STARTER_KIT.md); Book NN §X, Book MM §X.
- **Performance & tuning:** Book PP §X–§Y; [PATTERNS.md](PATTERNS.md) (named optimizations).
- **Debugging:** [SYMPTOMS.md](SYMPTOMS.md) → [ANTI_PATTERNS.md](ANTI_PATTERNS.md).
<!-- /EXAMPLE -->
